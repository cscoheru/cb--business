## 关键技术决策

### 1. 动态渲染 vs 静态生成

**决策**: 对需要实时数据的页面使用动态渲染

**原因**:
- 文章内容频繁更新
- 用户需要看到最新资讯
- 避免构建时调用 API 导致错误

**实现**:
```typescript
export const dynamic = 'force-dynamic';
```

### 2. 公开 API vs 认证 API

**决策**: 文章端点公开访问，触发爬取需要认证

**原因**:
- 免费用户可以浏览所有内容
- 只有高级功能（触发爬取）需要认证
- 符合 Freemium 商业模式

### 3. 类型系统对齐

**问题**: 前后端类型定义不一致导致运行时错误

**解决方案**:
- 后端: 正确转换数据库模型到响应模型
- 前端: 更新类型定义以匹配后端响应
- 统一处理 null 值和可选字段

## 已知问题和限制

### 1. 爬虫 SSL 证书问题

**错误**: `certificate verify failed`

**影响**: 无法爬取实际数据，目前使用测试数据

**临时方案**: 使用 MockAIProcessor 和手动创建的测试数据

**长期方案**:
- 配置正确的 SSL 证书
- 或使用不验证 SSL 的爬虫配置
- 或使用代理服务器

### 2. 国家筛选功能缺失

**问题**: 后端 Article 模型没有 country 字段

**影响**: 前端无法按国家筛选文章

**临时方案**: 只按地区筛选

**长期方案**: 添加 country 字段到数据库模型和 API

### 3. 认证流程未完成

**问题**: 登录/注册按钮链接到 `#`

**影响**: 用户无法注册和登录

**状态**: 后端认证 API 已实现，前端需要对接
